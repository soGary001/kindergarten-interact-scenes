# 幼儿园英语口语比赛互动 App — 设计文档

**日期：** 2026-05-31
**状态：** 已通过头脑风暴评审，待实现计划

---

## 1. 目标与范围

为一个国际幼儿园的英语口语比赛打造一个**现场放映 / 互动展示 App**。每个上台的孩子与一个可爱的 AI 角色完成一次简短互动：角色在场景里用英语提问，孩子看着场景内容开口作答（说给现场评委听），角色感谢孩子并送出一个幸运数字作为礼物。

App 打包成可在其他电脑上直接打开的 **Windows `.exe`** 和 **Mac `.dmg`**，且**不包含 / 不泄露任何 API Key**。

### 明确不做（YAGNI）

- ❌ 拼图、单词拼读（phonics）、造句延伸 —— 原需求第 3 点，已整体移除。
- ❌ 语音识别 / 自动评分 —— App 不"听"孩子说话。
- ❌ 录音 / 麦克风 —— App 不录制、不存储任何音频，无需麦克风权限。
- ❌ 孩子姓名 / 编号录入 —— 待机页只有一个「开始」按钮。
- ❌ 运行时联网 / 实时调用大模型 —— 全部内容离线预生成。

---

## 2. 核心流程（一个孩子一轮）

```
待机页（仅「开始」按钮）
   │  老师点「开始」
   ▼
随机选定：角色 + 该角色的一个丢失物品 + 一个地点
   │
   ▼
展示插画场景（物品贴图叠放在选中地点上）
自动播放角色英语提问语音："Where is my [item]?"
屏幕显示英文字幕
   │  孩子对着评委开口作答（App 静候，不录音、不评分）
   │  老师按「下一步」
   ▼
播放角色英语语音："Thank you! Here is your lucky number… [N]!"
撒花 / 弹跳动画展示 1–10 的幸运数字
   │  老师按「下一步」（或自动）
   ▼
回到待机页 → 下一个孩子
```

- **一个孩子 = 一个角色 + 一个问题 + 一个幸运数字。** 节奏快，适合人多。
- **随机选定**角色 / 物品 / 地点（每次"开始"重新随机）。
- **老师 / 工作人员用键盘或按钮推进节奏**（因为 App 不识别语音）。

## 3. 角色 / 物品 / 地点 内容模型

### 3.1 五个角色与丢失物品

| 角色 | 英文 | 丢失物品 |
|---|---|---|
| 老奶奶 | Grandma | glasses |
| 小男孩 | Boy | football, toys |
| 小女孩 | Girl | puppy, kitten |
| 爸爸 | Dad | keys, wallet, newspaper |
| 妈妈 | Mom | handbag, necklace, ring |

### 3.2 地点（来自原始需求清单）

- **老奶奶 / 爸爸 / 妈妈（客厅场景共用）：** table, TV cabinet, chair, sofa, windowsill, desk lamp, carpet, wardrobe, bookshelf（老奶奶额外含 pillow）
- **小男孩：** by the door, grass, shelf, chair, sofa, windowsill, carpet
- **小女孩：** station, park, garden, balcony, shelf, carpet

### 3.3 提问模式："找东西 · 说地点"

- 角色找不到某物品 → 问 `Where is my [item]?`（复数物品如 glasses 用 `Where are my glasses?`）。
- 期望孩子答 `It's on the [location].` / `They are on the [location].`
- 每个地点关联一个**自然介词**（on the table / on the sofa / **in** the wardrobe / **by** the door / **on** the grass 等），物品贴图按介词语义叠放（放在上面 / 里面 / 旁边）。
- 字幕只显示**问题**，不显示答案（答案是孩子的任务）。

### 3.4 场景背景复用与"锚点 + 贴图"渲染

- 共需约 **3 张场景背景插画**：客厅（覆盖老奶奶/爸爸/妈妈的全部地点 + pillow）、男孩场景（室内带一扇门通向草地）、女孩场景（室外为主 + 阳台 + 架子 + 地毯）。
- 每张背景图为每个地点定义一个**锚点坐标**（及对应介词 / 摆放方式）。
- 每个丢失物品做成**透明 PNG 贴图**，运行时叠放到选中地点的锚点上。
- 好处：任意"物品 × 地点"组合都能即时合成，无需为每个组合单独出图。

### 3.5 幸运数字

- 纯装饰性礼物，随机 **1–10**，配撒花 / 弹跳庆祝动画。
- 与任何后续玩法无关（拼图已移除）。

## 4. 技术架构

### 4.1 总体：离线预生成（关键决策）

把 Mimo 当成**构建期的"制作工具"**，而非运行时依赖：

- **构建时**（仅在开发机运行，用到 API Key）：用脚本生成全部台词文字 + TTS 语音 + 插画资源，存为静态文件。
- **运行时**（分发出去的 App）：纯离线播放器，**不含 API Key，不联网，零延迟**。

由此自动满足两个硬性要求：① App 里根本没有 Key，不可能泄露；② 现场不依赖网络，可预先审听 / 彩排，稳定可控。

### 4.2 桌面壳：Tauri

- 用 **Tauri**（Rust + Web 前端）打包，产物体积小（数 MB 级），一键出 `.exe` 与 `.dmg`。
- 前端用纯 HTML / CSS / JS（或轻量框架），无后端、无数据库。
- 无需麦克风等敏感权限。

### 4.3 构建脚本 `build-assets/`

输入一份**内容配置**（角色 × 物品 × 地点 × 台词模板 + 每地点的介词/摆放），脚本：

1. 调 **Mimo 文本模型**（`mimo-v2.5-pro` / `mimo-v2.5`）为每个角色生成多条**带人设**的台词变体：问候、提问、感谢 + 报幸运数字。每类多条以便轮换，降低重复感。
2. 调 **Mimo TTS**（建议用 `mimo-v2.5-tts-voicedesign` 为 5 个角色设计**各具特色的音色**：慈祥老奶奶 / 活泼男孩 / 甜美女孩 / 沉稳爸爸 / 温柔妈妈）把台词转成 mp3。
3. 生成插画资源：**场景背景 / 物品贴图 / 角色形象**（见 §7 待落实项）。
4. 产物落到 `app/assets/`，再 `tauri build` 出双平台安装包。
5. API Key 仅存于脚本运行环境（`.env`，加入 `.gitignore`，不进 git、不进包）。

### 4.4 运行时数据

App 内置一个 `content.json` 清单：每个角色有哪些物品、哪些地点、各台词变体对应的音频文件名、各场景背景与锚点坐标、物品贴图文件名。运行时据此随机组合并渲染。

## 5. 现场操作流程与交互

| 步骤 | 屏幕 | 操作 |
|---|---|---|
| 1 | 待机页（标题 + 大「开始」按钮 + 散落几何动画） | 老师点「开始」 |
| 2 | 场景提问页：角色 + 插画场景（物品在地点上）+ 气泡英文字幕，自动播放提问语音 | 孩子作答，老师按「下一步」 |
| 3 | 幸运数字页：角色感谢语音 + 撒花 + 大号数字 1–10 | 老师按「下一步」回待机 |

**键位：**
- `空格` / `回车` / `→` = 下一步
- `Esc` = 回待机页
- 屏幕同时提供大号按钮，鼠标 / 触屏可点。

---

## 6. 视觉 / UX 设计方向

- **风格：** 孟菲斯（Memphis）+ 马卡龙粉彩背景 + 散落几何形状（圆点、三角、波浪、Z 字），可爱、抓眼球、多巴胺。
- **元素：** 圆润大按钮（带硬阴影 offset）、气泡台词框、撒花 / 弹跳 / 缩放等欢快动画。
- **插画：** 角色与场景最终用 **AI 生成插画**（构建期产出），风格统一为扁平可爱卡通。
- **语言：** 角色**全程英语**；屏幕**显示英文字幕**（问题文本）。
- **适配：** 面向投影 / 大屏全屏放映，大字号、高可读性。

## 7. 待落实项 / 风险

1. **生图工具（必须解决）：** 用户提供的 Mimo 模型列表为文本 + TTS，**不含图像生成模型**。需在构建阶段确定插画来源：可用的 AI 生图工具、或用户提供生图模型/Key、或退而求其次用矢量插画素材。因插画为构建期一次性产出的静态图，不影响 App 运行时架构。
2. **音色一致性：** voicedesign 生成的角色音色需在所有台词间保持一致（建议同一角色用同一音色配置批量生成）。
3. **介词正确性：** 每个地点的介词 / 物品摆放需人工校对，确保英语自然（in the wardrobe / on the carpet / by the door / on the grass）。
4. **台词变体数量：** 每角色每类台词生成的变体数（建议每类 ≥3 条）需与音频体积权衡。
5. **双平台签名：** `.exe` / `.dmg` 未签名时系统可能弹安全警告；非公开使用下可接受，需告知操作者放行方式。

## 8. 资源清单（估算）

- 场景背景插画：约 3 张
- 角色形象：5 个
- 物品贴图（透明 PNG）：glasses, football, toys, puppy, kitten, keys, wallet, newspaper, handbag, necklace, ring —— 共 11 个
- TTS 音频：5 角色 ×（问候/提问/感谢报数）× 变体数 × 物品/数字组合（提问按物品、感谢按数字 1–10 复用），数量可控
- 装饰几何图形：少量可复用 SVG / CSS
